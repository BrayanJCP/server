Desarrollar Front-End (Web)

Para adquirir los conocimientos básicos de un desarrollador front-end se debe tener claro los conceptos de html, css y javascript:

  • HTML: HyperText Markup Language (lenguaje de marcas de hipertexto). Básicamente es un documento web (.html), el cual contiene una serie de etiquetas que reconocen los navegadores(Chrome, Firefox, Internet Explorer, entre otros) para organizar la información(Por lo general son textos e imagenes).

  • CSS: es un mecanismo simple para añadir estilos a un documento web(HTML o XML).

  • Javascript: Es un lenguaje de programación que permite hacer el procesamiento de la información.

Patrones de diseño

Los patrones de diseño son soluciones reutilizables a problemas comunes que surgen durante el diseño y desarrollo de software. Representan buenas prácticas desarrolladas para aplicarse en diferentes proyectos. Se pueden utilizar estos patrones como guías para resolver problemas específicos de manera eficiente y estructurada.

Patrones Creacionales

  • Abstract Factory: Crea familias de objetos relacionados sin especificar clases concretas.
  • Builder: Construye objetos complejos con representaciones diferentes.
  • Factory Method: Define interfaz para crear objetos, subclases deciden instanciación.
  • Prototype: Crea objetos copiando un prototipo existente.
  • Singleton: Asegura una única instancia de una clase.

Patrones Estructurales

  • Adapter: Permite que interfaces incompatibles trabajen juntas.
  • Bridge: Desacopla abstracción de implementación para variar independientemente.
  • Composite: Crea estructuras de árbol para jerarquías parte-todo.
  • Decorator: Añade responsabilidades dinámicamente a un objeto.
  • Facade: Proporciona interfaz unificada a un subsistema.
  • Flyweight: Comparte objetos para soportar grandes cantidades eficientemente.
  • Proxy: Controla el acceso a otro objeto.

Patrones de Comportamiento

  • Chain of Responsibility: Pasa solicitudes a través de una cadena de objetos.
  • Command: Encapsula una solicitud como un objeto.
  • Interpreter: Define representación para la gramática de un lenguaje.
  • Iterator: Accede secuencialmente a elementos de un objeto agregado.
  • Mediator: Define cómo interactúan un conjunto de objetos.
  • Memento: Captura y restaura el estado interno de un objeto.
  • Observer: Define dependencia uno-a-muchos entre objetos.
  • State: Altera comportamiento de un objeto según su estado interno.
  • Strategy: Define familia de algoritmos intercambiables.
  • Template Method: Define esqueleto de algoritmo, subclases implementan pasos.
  • Visitor: Define nueva operación sin modificar clases de objetos.
